nastaveni aby prikaz get pri stahovani z netu v cmd sel pres nastavene proxy

	sudo nano /etc/wgetrc
	odkomentovat 3 radky s proxy a nastavit proxy serveru

	https_proxy = http://172.16.1.10:3128/
	http_proxy = http://172.16.1.10:3128/
	ftp_proxy = http://172.16.1.10:3128/

	kontrola:
	wget fileurl

	http://stackoverflow.com/questions/11211705/setting-proxy-in-wget

nastaveni proxy, update a upragde
	
	vytvorit soubor 10proxy v adresari /etc/apt/apt.conf.d/
	a vlozit do nej proxy serveru
	
	cd /etc/apt/apt.conf.d
	sudo nano 10proxy
	

	Acquire::http::Proxy "http://172.16.1.10:3128/";
	Acquire::ftp::Proxy "http://172.16.1.10:3128/";
	Acquire::https::Proxy "http://172.16.10:3128/";
	
	sudo apt-get update
	sudo apt-get upgrade


nastaveni casu ze serveru
	najit soubor /etc/ntp.conf zakomentovat servery a napsat vlastni
	
	sudo nano /etc/ntp.conf

	server 172.16.1.2 iburst
	
	kontrola:
	sudo /etc/init.d/ntp restart

nastaveni staticke ip eth, wifi
		
	dhcp napsat nakonec tohle do
	
	sudo nano /etc/dhcpcd.conf
	
	interface eth0
	static ip_address=172.16.4.85
	static domain_name_servers=172.16.1.10



		naposledy nejak neslo:	 
		sudo nano /etc/network/interfaces
	
		pridam, zamenim manual za static
		iface wlan0 inet static
        	address 10.1.1.1
        	netmask 255.255.0.0
        	gateway 172.16.1.10


	http://raspberrypi.stackexchange.com/questions/37920/how-do-i-set-up-networking-wifi-static-ip-address
